# Compliance Canary

Tracking how ready the crypto sector is for federal stablecoin and crypto regulation — with data, not opinion.

**[View the live dashboard →](https://jschulman.github.io/compliance-canary)**

## What Is This?

An open-source dashboard that aggregates publicly-available job posting data to answer one question: **Is the crypto sector actually staffing up for federal regulation?**

The **Regulatory Readiness Signal** counts mentions of compliance credentials and regulatory keywords across all open crypto-native middle-market finance roles: CPA, Big 4 audit experience, SOX, PCAOB, MSB/MTL state licensing, FINRA/SEC supervisory experience, GAAP/ASC technical accounting, and GENIUS Act readiness. A rising signal means crypto-native CFOs are hiring people who can pass a federal audit. A flat or declining signal means the sector is betting regulation won't come for them.

| Total signals | Phase | What It Means |
|-------|-------|---------------|
| 0–10 | Asleep | Sector is not hiring for regulation; no demand signal |
| 11–30 | Stirring | Early hires for compliance; concentrated at a few issuers |
| 31–75 | Alert | Real, distributed demand for regulatory-ready finance staff |
| 76+ | Mobilized | Crypto is staffing up to meet federal oversight at scale |

## Signals

| Signal | Source | Frequency | What It Measures |
|--------|--------|-----------|-----------------|
| Total Signal Count | Greenhouse / Ashby / Lever public job-board APIs | Daily | Total mentions of tracked credentials across all open finance JDs |
| Per-Credential Count | Greenhouse / Ashby / Lever | Daily | Mention count per credential (CPA, SOX, PCAOB, MSB-MTL, FINRA-SEC, GAAP-ASC, GENIUS Act) |
| Tracked JD Count | Greenhouse / Ashby / Lever | Daily | Total open Tier-1/2/3 finance JDs in the corpus (the denominator) |
| Signal Over Time | Computed from snapshots | Daily | Time series of total readiness signal mentions |

Every signal uses **free, public data sources** — no proprietary APIs, no paywalls.

## Design Principles

- **Data, not opinion.** The dashboard presents signals. It does not editorialize.
- **Reproducible.** Methodology is open. Anyone with a list of crypto-native companies and access to their public careers pages can compute these numbers.
- **Automated.** Updated daily on a schedule. No human curation of the daily data.
- **Durable.** Only signals with reliable, ongoing public sources are included.
- **Accessible.** Single page. No login. No paywall. Mobile-friendly.

## Architecture

```
data/snapshots/         Versioned daily JSON snapshots
data/latest.json        Most recent snapshot (what the dashboard reads)
docs/                   Static dashboard (HTML/JS/CSS — GitHub Pages)
.github/workflows/      Automated refresh schedule
METHODOLOGY.md          How every signal is computed, with sources
```

Data is generated by scanning a curated list of crypto-native middle-market companies' public job-board APIs (Greenhouse, Ashby, Lever), then keyword-matching JD bodies against a fixed list of credential and regulatory terms. The target list and aggregation code are maintained privately; only aggregate metrics are published here. See [METHODOLOGY.md](METHODOLOGY.md) for details.
