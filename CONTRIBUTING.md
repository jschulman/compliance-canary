# Contributing

Compliance Canary is a single-author project maintained by Jay Schulman. It's open-source so that anyone can verify the methodology and rebuild the dashboard from scratch if they want to — not because it's accepting external code contributions.

## What's welcome

- **Issues** that flag genuine methodology problems: a missing credential we should be tracking, a regex that's matching too loosely, a bug in the dashboard's rendering. Be specific.
- **PRs** for typo fixes, accessibility improvements to the dashboard, or documentation clarifications.
- **Discussion** about which regulatory signals deserve to be on the tracked list.

## What's not welcome

- PRs that change the metric definition without prior discussion in an issue. The metric is intentionally simple.
- PRs that add new external dependencies. The dashboard is intentionally static.
- Feature requests for individual-company breakdowns. The project is corpus-aggregated by design.

## How the data is generated

The data file `data/latest.json` and the per-day snapshots in `data/snapshots/` are generated by a private pipeline that scans public job-board APIs (Greenhouse, Ashby, Lever) and aggregates them through the credential taxonomy in [METHODOLOGY.md](METHODOLOGY.md). The pipeline's source is not in this repo. If you want to reproduce the data, follow the methodology — don't reverse-engineer the JSON.
